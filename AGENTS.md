# RapidKL Bus Dashboard

Live bus tracking dashboard for RapidKL. PWA for iPhone home screen.

## Architecture (v9)

- **Hosting**: GitHub Pages
- **Live data**: Prasarana Socket.io AVL (`rapidbus-socketio-avl.prasarana.com.my`)
  - JSON over WebSocket, gzip-compressed via pako
  - Session token from public kiosk
  - Smart connect: connects on page focus, disconnects on blur
- **Static data**: Pre-built `data/static.json.gz` (2.9 MB)
  - Generated by `scripts/build-static.js` from `api.data.gov.my` GTFS static ZIPs
  - Refreshed weekly via GitHub Action (`.github/workflows/refresh-static.yml`)
  - Loaded same-origin, decompressed with pako, cached in localStorage (7 days)
- **Stack**: Vanilla JS, Leaflet, Socket.io v2, Pako (all CDN)
- **Poll interval**: 5 seconds (live push via socket)
- **PWA**: manifest.json + service worker + apple-touch-icon

## Key Features

- **Live ETA**: Calculated from bus GPS distance along route shape to selected stop
- **Quick Dial**: Save favorite route+stop combos; auto-restores last used on revisit
- **Route filtering**: Select a route → only that route's buses on map
- **Bus popups**: Tap marker → plate number, GPS time, speed
- **Minutes display**: Clock shows minutes until arrival, not wall-clock time

## API Notes

- Socket.io requires a session ID — obtained from the public kiosk page
- GTFS-R trip updates not yet available from api.data.gov.my (planned 2026)
- Live ETA is GPS-based, not trip-update-based
- Some trip IDs in realtime feed don't match static schedule — route matching handles this
